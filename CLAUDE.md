# Chi's JobSearcg Claude Code Preferences
This file provides guidance to Claude Code (claude.ai/code) when working with content in job-search repository.

## Structure

- `original/` — read-only source files; contains `resume.md` (professional CV) and `job-description-template.md` (template to duplicate for new JD files); do not edit any file in this folder.
- `job-description` - job description.
- `.obsidian/` — Obsidian app configuration (appearance, plugins, workspace state); do not edit these files manually.

## Working with Content

All files are plain markdown (`.md`). When editing markdown files:
- Preserve existing heading hierarchy and whitespace formatting
- Resume.md uses a flat structure with bold role titles, company links, and bullet-point achievements
- Use fact and numbers in `resume.md`. 1 phase = 1 fact. Do not skip the phrase.
- Fetch the complete, full, unabridged text of the job posting without summarization, without modification. Do not summarize, condense, or omit any section. Include every word from every section exactly as it appears on the page.
